dotenv 관련

우선순위

dotenv 파일에는 읽어오는 우선순위가 있다.

ex)

  • cra에서는 링크와 같이 우선순위를 두고 있다
    • .env: Default.
    • .env.local: Local overrides. This file is loaded for all environments except test.
    • .env.development, .env.test, .env.production: Environment-specific settings.
    • .env.development.local, .env.test.local, .env.production.local: Local overrides of environment-specific settings.
  • vite 요기

라이브러리 별로 다른지, 기본적으로 dotenv 설정되어있는지는 잘 모르겠다.

라이브러리 별 컨벤션

dotenv 파일에서 커스텀 변수명 앞에 붙는 글자들은 라이브러리에서 권장하는 컨벤션이다.

ex) VITE, REACT_APP, NEXT

cra에서 REACT_APP을 붙여주지 않으면 안된다.(참고)

dotenv-expand

dotenv-expand를 사용해서 .env안에서 변수를 선언해서 사용할 수 있다.

ex)

PASSWORD="s1mpl3"
DB_PASS=$PASSWORD

Gatsby-remark

gatsby-remark의 graphql에서 excerpt를 사용하면 본문 내용을 잘라서 가져올 수 있다.

ex)

export const pageQuery = graphql`
  {
    allMarkdownRemark(sort: { frontmatter: { date: DESC } }, limit: 6) {
      edges {
        node {
          fields {
            slug
          }
          frontmatter {
            title
            path
            date
            tags
          }
          excerpt(pruneLength: 150, truncate: true, format: PLAIN)
        }
      }
    }
  }
`;

Vite-bundle-visualize

vite에서도 번들링된 파일을 분석하고 싶으면 해당 라이브러리를 사용하면 된다. 보여지는 형태는 ==webpack-bundle-analyzer==와 비슷하다.

Sentry error sampling rate

센트리를 사용하면 에러에 대한 정보를 얻을 수 있다. 하지만 너무 많은 양은 문제가 될 수 있어서 적절한 양이 필요한데, 이를 위해서 rate를 설정한다.

ex) 아래와 같이 rate를 0.25로 설정하면 25%의 에러를 전달한다.

Sentry.init({ sampleRate: 0.25 });

그 외에도 많은 정보에 대해서 sampling rate를 설정할 수 있다.(참고)
